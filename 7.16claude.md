## 知识点整理

### 1. `ref()` 和响应式
- `ref` 让变量变成"响应式"，变了页面自动更新
- 模板里直接用 (`{{ count }}`)，script 里要 `.value` (`count.value`)

### 2. Composable（组合式函数）
- 把可复用逻辑抽成函数，到处用
- 例：`useStorage.ts` 封装 localStorage，页面不用手动调 `getItem/setItem`

### 3. `getStorageData` 原理
- 创建 ref → 从 localStorage 读初始值 → watch 监听变化自动写回
- 之前 bug：在函数内部调用，每次新建 ref（局部变量），跟页面不同步
- 修复：在 `<script setup>` 顶层调用一次，全局共用一个 ref

### 4. Toast（vue-sonner）
- 替代 `alert()` / `confirm()`
- `app.vue` 放 `<Toaster />` 容器
- 页面里 `import { toast } from 'vue-sonner'` → `toast.success('消息')`

### 5. Nuxt 3 → Nuxt 4 升级
- 版本升到 4.4.8
- 目录结构：`app/app.vue`、`app/pages/`、`app/composables/`
- 关闭 SSR：`ssr: false`（解决 localStorage 服务端报错）
- `@/` 指向 `srcDir`（根目录或 `app/`）

### 6. CSS 变量设计系统
- 统一色板、间距、圆角、阴影
- `var(--color-primary)` 代替散装 `#1890ff`
- 改一个变量所有组件跟着变

### 7. 组件化
| 组件 | 核心功能 |
|------|---------|
| `AppCard` | 白色卡片容器 + variant（默认/outlined） |
| `AppTextarea` | 包装 textarea + 实现 v-model 双向绑定 |
| `AppButton` | 按钮 + variant/loading/disabled |

### 8. Props + Emit
- **Props**：父→子传参（`defineProps`）
- **Emit**：子→父发通知（`defineEmits`）
- `v-model` 展开：`:modelValue` + `@update:modelValue`
- `$attrs`：父组件没声明接收的参数，自动落到子元素上

### 9. v-model 原理
```vue
<!-- 等价写法 -->
<AppTextarea v-model="message" />
<AppTextarea :modelValue="message" @update:modelValue="message = $event" />
```

### 10. `editSource` 标记来源
- 用字符串 ref 标记当前编辑内容来自 AI 还是词库
- 模板根据来源显示不同标题

### 11. 布局演进
1. 顶栏导航 → 2. 两栏布局在 index.vue → 3. 侧边栏移到 app.vue 共享
- 共享数据用 composable（`useHistory`）
- cross-page 通信用 `window.dispatchEvent(CustomEvent)`

### 12. 提示词工程
- `---PROMPT_START---` / `---PROMPT_END---` 包裹提取精简版
- 后端 system prompt 控制 AI 输出格式（先分析再包裹）
