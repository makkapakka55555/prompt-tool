## 今日完成

### 1. 分类浏览改路由参数
- **问题**：分类浏览在首页靠 `activeCategory` provide 实现，切换分类时侧边栏"生成"高亮不消失
- **改法**：点击分类 → `router.push('/library?category=xxx')`
- **效果**：`/library?category=code`，URL 显示路径，分类过滤走 query
- `app.vue` 删掉 `activeCategory` ref/provide，分类链接改为直接 router.push
- `index.vue` 删掉浏览模式（`v-else` 整段）
- `library.vue` 加 `route.query.category` 过滤

### 2. 侧边栏逻辑
- 分类浏览区域 `v-if="$route.path === '/library'"`（只在词库页显示）
- 分类高亮改为 `$route.query.category === 'xxx'`
- 删掉 `selectedItem` provide / `pickFromLibrary`

### 3. 历史记录恢复（时序问题修复）
- **问题**：`dispatchEvent` 在 `router.push()` 之前触发，组件还没挂载，事件没人收
- **改法**：`localStorage.setItem('restore-data', data)` → 导航 → `onMounted` 时读取并删除
- 删掉 `provide/inject` / `watch` / `dispatchEvent`
- 支持点击 AI 消息也恢复（根据 index 找配对）

### 4. 参数面板
- 输入框和生成按钮之间加折叠面板（`<details>`）
- 温度（range 0-1，step 0.1）
- 最大 Token（number input）
- 参数已有变量 `temperature` / `maxTokens`，不需要新增

### 5. 边界情况处理
- **请求中按钮禁用**：`:disabled="loading"` ✅
- **网络超时重试**：`$fetch` 加 `retry: 1`，ofetch 自动重试 1 次
- **生成中途取消**：`AbortController` + 取消按钮
  - `send()` 开头创建 `abortController.value = new AbortController()`
  - `$fetch` 传 `signal: abortController.value.signal`
  - 取消时调 `abort()` → 抛 `AbortError` → catch 里静默退出

## 知识点

### AbortController
```javascript
const controller = new AbortController()
// signal → 传给 $fetch，监听中断信号
// abort() → 发中断信号
// 中断时 $fetch 抛 AbortError
```

### ofetch ($fetch) retry
```javascript
$fetch(url, { retry: 1 })
// 失败后自动重试 1 次，总共最多 2 次
```

### localStorage 跨页面传数据
```javascript
// source 存
localStorage.setItem('key', JSON.stringify(data))
// target 读
const raw = localStorage.getItem('key')
const data = JSON.parse(raw)
localStorage.removeItem('key')
```
比 `dispatchEvent` / `provide/inject` 更可靠，不受组件生命周期影响。
